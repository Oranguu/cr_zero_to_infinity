# 技术美术软件工程升级计划（Ray Chen 定制版）

目标：
从“能写小插件的TA” → “能主导团队级工具系统的工程架构师”

核心理念：
1. 先理解系统，再学语法
2. 先做小闭环，再做分布式
3. 每一阶段都要产出真实可用项目

---

# 阶段一：软件工程思维入门（1-2个月）

目标：理解“一个完整软件系统”是什么

## 1. 必须掌握的基础概念

- 什么是前端？
- 什么是后端？
- 什么是服务器？
- 什么是数据库？
- 什么是HTTP？
- 什么是API？
- 什么是客户端-服务器架构？
- 什么是本地应用 vs Web应用？

推荐学习方式：
- 看一套 Web 全栈入门课程（快速过一遍）
- 不追求深度，只追求理解结构图

---

## 2. 实战项目1：构建一个本地 Python 工具

技术栈：
- Python
- Tkinter / PyQt
- 文件读写
- JSON

项目示例：
- 本地资产统计工具
- 场景命名规范检测工具
- 一键生成项目周报工具

目标能力：
- 理解程序入口
- 理解模块拆分
- 理解数据结构
- 理解代码组织

---

# 阶段二：网络与API（1-2个月）

目标：理解“程序如何和外部世界通信”

## 必须掌握的概念

- HTTP 请求（GET / POST）
- JSON 数据格式
- RESTful API
- Token 鉴权
- 状态码（200 / 404 / 500）

---

## 实战项目2：做一个 API 调用工具

技术栈：
- Python requests
- 调用 OpenAI API
- 调用 GitHub API
- 调用 Notion API

项目示例：
- 自动生成周报
- 自动整理竞品资料
- 自动分析舆情

目标能力：
- 发送请求
- 解析返回数据
- 错误处理
- 封装 API 模块

---

# 阶段三：后端开发（2个月）

目标：你要能自己写一个“服务器”

技术栈：
- Python
- Flask 或 FastAPI
- 简单数据库（SQLite / MySQL）

你要理解：

- 路由（route）
- 请求处理流程
- 数据库存储
- 用户系统
- 会话管理

---

## 实战项目3：构建一个内部工具平台

示例：
- 团队资产管理平台
- 美术评价收集平台
- Skill 仓库系统

能力目标：
- 构建接口
- 连接数据库
- 多人访问
- 基础权限系统

---

# 阶段四：前端入门（1-2个月）

目标：理解前端如何展示数据

技术栈建议：

初学版本：
- HTML
- CSS
- JavaScript

进阶版本：
- React
- Vue

你要理解：

- DOM
- 组件
- 状态管理
- 前后端数据交互

---

## 实战项目4：给阶段三的系统做一个前端界面

目标：
- 理解前后端分离
- 能让别人通过浏览器访问你的工具

---

# 阶段五：部署与服务器（1个月）

目标：让你的系统真正在线上运行

你要理解：

- 什么是云服务器
- 什么是域名
- 什么是Docker
- 什么是Nginx
- 什么是HTTPS

实践：
- 把你的工具部署到云服务器
- 团队成员可以访问
- 解决真实问题

---

# 阶段六：架构思维升级（长期）

开始研究：

- MVC架构
- 微服务
- 消息队列
- 缓存
- 并发
- 性能优化
- 日志系统
- 监控系统

---

# 最终能力模型

你将具备：

1. 本地工具开发能力
2. API整合能力
3. 后端系统构建能力
4. 基础前端能力
5. 云部署能力
6. 架构思维

你将可以：

- 主导一个AI工具平台
- 设计Agent执行系统
- 构建团队级自动化平台
- 真正拥有“工程化落地能力”

---

# 关键建议

1. 每个阶段必须做真实项目
2. 不要只看视频
3. 用AI辅助，但不能替你思考
4. 每周必须写代码
5. 所有项目都围绕你真实业务

---

终极目标：

从“会写工具的人”
进化为“构建系统的人”
