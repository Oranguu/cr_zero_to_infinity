# 阶段一：软件工程思维入门（1-2个月）

目标：理解“一个完整软件系统”是什么

## 1. 必须掌握的基础概念

下面这些概念不用背，先建立“脑子里有一张结构图”即可。以后做项目时会反复碰到，自然会熟。

---

**什么是前端？**

前端就是“用户看得见、摸得着”的那一层。在网页里，就是你在浏览器里看到的页面、按钮、表单、动画；在桌面软件里，就是你用 Tkinter / PyQt 做出来的窗口和控件。前端负责两件事：**展示信息** 和 **接收用户操作**。作为 TA，你做过的美术插件界面，本质上就是某种“前端”；区别在于，Web 前端用 HTML/CSS/JavaScript 在浏览器里跑，桌面前端用各语言自带的 GUI 库在本机跑。

---

**什么是后端？**

后端是“在背后干活”的那一层。用户点了“提交”“查询”“保存”，前端把请求发出去，真正做计算、查数据、写数据库、调第三方服务的是后端。后端不负责画界面，只负责**业务逻辑**和**数据读写**。例如：你写一个“团队资产管理平台”，前端负责列表、筛选、上传按钮；后端负责把资产信息存进数据库、按条件查询、做权限校验。前后端通过“接口”（见下文 API）通信。

---

**什么是服务器？**

服务器有两层意思，容易混，分开记：

1. **硬件层面**：一台（或一组）常年开机、连在网上的电脑，专门用来跑你的后端程序、数据库等，别人通过网络访问它。公司机房里的机器、云厂商的“云服务器”都是这种。
2. **软件层面**：我们常说“起一个服务器”“跑一个 server”，指的是**正在运行的后端程序**——它监听某个端口，等待别人发请求过来并处理。例如用 Flask 写几行代码跑起来，你的电脑就在本机充当了一台“服务器”。

开发时经常是“本机当服务器”；上线后，服务器程序会跑在云服务器或公司机器上。

---

**什么是数据库？**

数据库是**持久化存数据**的地方。程序重启、电脑关机，数据还在。你写的 JSON 文件也算一种“存数据”，但规模一大、多人同时读写就不合适了；数据库专门解决：**结构化存储**（表、字段）、**查询**（按条件查）、**并发**（多人同时用不出错）。常见的有 SQLite（单文件、适合小工具）、MySQL/PostgreSQL（适合正式服务）。先理解“数据库 = 存业务数据的地方，由后端来读写”即可。

---

**什么是 HTTP？**

HTTP 是一种**网络通信协议**，约定“客户端和服务器之间怎么说话”。你打开网页、点链接、提交表单，浏览器和服务器之间大多就是用 HTTP 在传数据。典型过程：客户端发“请求”（我要这个 URL 的资源 / 我要用这些数据调用你的接口），服务器回“响应”（这是页面内容 / 这是接口返回的 JSON）。记住：**请求-响应、一问一答**。后面学 API 时，你会反复和 HTTP 打交道（GET、POST、状态码等）。

---

**什么是 API？**

API（Application Programming Interface）即**应用程序编程接口**，就是“别人写好的一段能力，你按约定方式调就能用”。对前端来说，调用后端的接口（例如“提交一条资产记录”）就是在调 API；对你写的 Python 脚本来说，调用 OpenAI、GitHub、Notion 的接口也是在调 API。**接口 = 约定好的入口 + 请求格式 + 返回格式**。不用关心对方内部怎么实现，只要按文档发请求、解析返回数据即可。你后面阶段二会大量做“调外部 API”的练习。

---

**什么是客户端-服务器架构？**

一种常见分工方式：**客户端**（前端、桌面程序、手机 App）负责和用户交互、发请求；**服务器**（后端程序跑在服务器上）负责处理请求、查改数据、返回结果。双方通过网络（HTTP 等）通信。你做的本地小工具可能只有“客户端”，没有服务器；一旦要做“多人共用、数据集中存”的系统，就会自然变成客户端-服务器架构。

---

**什么是本地应用 vs Web 应用？**

- **本地应用**：程序装在你电脑上，界面用本机 GUI（如 Tkinter），数据多在本地文件或本机数据库，不依赖浏览器。你阶段一做的 Python 小工具就是典型本地应用。
- **Web 应用**：用浏览器打开，前端是 HTML/CSS/JS，后端在服务器上，数据在服务器端的数据库里；多人通过浏览器访问同一个地址即可使用。团队级工具、内部平台通常是 Web 应用。

很多系统是“Web 前端 + 后端 API”，即：界面在浏览器里，逻辑和数据在服务器上，这就是前后端分离的形态，你后面阶段三、四会实践到。

---

**推荐学习方式**

- 看一套 Web 全栈入门课程（快速过一遍）
- 不追求深度，只追求：能在脑子里画出一张图——用户 → 前端 → HTTP → 后端 → 数据库，以及“本地应用 vs 浏览器里打开的 Web 应用”的区别
